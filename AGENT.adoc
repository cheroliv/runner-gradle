== Projet

**Nom** : engine — point d'entree unique du workspace, build script only
**Stack** : Kotlin DSL Gradle, `includeBuild()` pour plugins N0/N1/N2 en dev mode
**Role dans le DAG** : N3 — Orchestration (voir `AGENT_GOVERNANCE.adoc` → Regle 4bis)
**N'EST PAS** un plugin Gradle — pas de `src/main/kotlin`, pas de `publishToMavenLocal`

Le projet `engine/` orchestre la flotte de plugins Gradle (N0→N2) depuis un seul
`build.gradle.kts`. Il sert d'interface CLI pour le developpeur et d'API pour le SaaS Edster.

Consomme les plugins de `foundry/` comme dependances ou `includeBuild`, lit les secrets
depuis `configuration/` (cercle 1), et applique les regles de gouvernance globale.

== Regles Absolues

=== Git

**INTERDICTION FORMELLE** de commit/push/merge sans permission explicite de l'utilisateur.

Commandes autorisees (lecture seule) : `git status`, `git diff`, `git log`, `git show`
Commandes interdites (sauf ordre) : `git add`, `git commit`, `git push`, `git merge`, `git rebase`

=== Pas de `publishToMavenLocal`

Engine n'est pas un plugin. Il n'y a rien a publier. La seule commande Gradle legitime
est l'execution de taches (`./gradlew aggregateGraphs`, `./gradlew verifyDagAcyclic`, etc.).

=== Securite

* Secrets lus depuis `configuration/` (cercle 1), jamais stockes ici
* Zéro credential dans `build.gradle.kts`

=== Tests

L'agent NE DOIT JAMAIS lancer de tests lors de la procedure de fin de session SANS permission explicite.

== Procedure de Fin de Session (obligatoire)

. Lire `.agents/SESSION_PROCEDURE.adoc`
. Executer les 6 etapes standard pour `foundry/engine/`

== Contrat DAG — N3 Orchestration

Engine est le seul projet N3. Regles absolues du DAG (voir `AGENT_GOVERNANCE.adoc` Regle 4bis) :

* *Peut importer* : tous les N0 (`graphify-gradle`), N1 (`codebase-gradle`), N2 (`plantuml-gradle`, `slider-gradle`, `bakery-gradle`, etc.)
* *Ne peut JAMAIS etre importe* : par N0, N1, ou N2. Aucun plugin du workspace ne depend d'engine.
* *Est le consommateur du graph.json global* : engine agrege tous les `graph.json` locaux en un graphe unifie

== Taches Cibles

Les taches Gradle a construire dans `build.gradle.kts` :

[cols="1,3,3"]
|===
| Tache | Description | Depend de
| `aggregateGraphs` | Agrege tous les `graph.json` locaux des projets N0/N1/N2 en un graphe global | `graphify-gradle`
| `verifyDagAcyclic` | Verifie qu'aucun plugin n'importe un plugin de niveau superieur | Aucune (lecture `build.gradle.kts`)
| `provisionWorkspace` | Provisionne un workspace client complet (tous les plugins + office/ + configuration/) | Tous les plugins
| `augmentOpencode` | Pipeline complet walk→index→query→format → `/tmp/opencode-context.txt` | `codebase-gradle` EPIC 9
|===
